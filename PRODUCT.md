# Что делает приложение

Сервис отслеживает новые посты в заданном списке Telegram-каналов и автоматически пересылает их в ваш единый канал-агрегатор.

## Стек технологий

- **Язык:** TypeScript (Node.js)
- **Бот:** Библиотека grammY (Bot API)
- **Сбор данных:** HTTP-скрапинг публичных страниц каналов (t.me/s/{channel})
- **Хранение состояния:** JSON-файл с последними ID сообщений по каждому каналу

## Архитектура

Приложение работает в рамках одного процесса и использует polling-подход:

### Планировщик (Scheduler)

- Запускается по таймеру каждые 5 минут (настраивается через `POLL_INTERVAL_MS`)
- Читает список каналов из файла `channels.txt`
- Для каждого канала запрашивает HTML-страницу `https://t.me/s/{channel}`
- Парсит ID постов из HTML и сравнивает с сохранённым состоянием (`channel-state.json`)
- Определяет новые посты и передаёт их на пересылку

### Бот (Bot)

- Авторизуется через токен от @BotFather
- Является администратором в вашем канале-агрегаторе
- Выполняет `forwardMessage` для каждого нового поста
- Обрабатывает команды управления каналами (`/add_channel`, `/remove_channel`, `/list_channels`)

### Хранилище каналов (Whitelist Store)

- Файл `channels.txt` содержит список username каналов (по одному на строку)
- Управляется через команды бота в Telegram
- Изменения применяются без перезапуска сервиса

### Хранилище состояния (State Store)

- Файл `channel-state.json` хранит последний обработанный ID сообщения для каждого канала
- Используется для определения новых постов между циклами polling

## Ключевые преимущества решения

- **Не требуется аккаунт пользователя:** Достаточно только бот-токена
- **Простота настройки:** Нет необходимости в MTProto, API_ID, API_HASH или StringSession
- **Безопасность:** Бот работает через официальный Bot API
- **Гибкое управление каналами:** Добавление и удаление каналов через команды в Telegram
- **Устойчивость к перезапускам:** Состояние сохраняется в файл и восстанавливается при старте

## Решения по дизайну

| Решение          | Выбор                                                                     |
| ---------------- | ------------------------------------------------------------------------- |
| Метод публикации | `forwardMessage` — пересылка с сохранением атрибуции оригинального автора |
| Фильтрация       | Без фильтрации — все новые посты пересылаются без исключений              |
| Каналы-источники | Управление через команды бота, хранение в `channels.txt`                  |
| Маршрутизация    | Единый канал-агрегатор — все посты из всех источников идут в одно место   |
| Интервал опроса  | 5 минут (настраивается)                                                   |
