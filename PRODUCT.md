# Что делает приложение

Сервис в реальном времени отслеживает новые посты в заданном списке Telegram-каналов (включая чужие закрытые/открытые каналы) и автоматически пересылает их в ваш единый канал-агрегатор.

## Стек технологий

- **Язык:** TypeScript (Node.js)
- **Чтение (MTProto):** Библиотека GramJS (клиент для работы от лица пользователя)
- **Публикация (Bot API):** Библиотека grammY (клиент для классического бота)

## Архитектура: Разделенная схема (Producer-Consumer)

Приложение работает в рамках одного процесса, но логически разделено на две независимые роли:

### Читатель / Producer (Технический аккаунт)

- Авторизуется один раз через StringSession (сохраняется в .env)
- Подписан на целевые каналы-источники
- Работает в стелс-режиме (read-only), получая push-уведомления (NewMessage) от серверов Telegram через исходящее TCP-соединение
- Перехватывает `message_id` нового поста и передает его Писателю

### Писатель / Consumer (Официальный бот)

- Авторизуется через токен от @BotFather
- Является администратором в вашем канале-агрегаторе
- Получает данные от Читателя и делает публикацию (`forwardMessage`) через Long Polling

## Ключевые преимущества решения

- **Нулевой риск блокировки:** Технический номер используется исключительно для чтения, что не вызывает подозрений у антиспам-алгоритмов Telegram
- **Без открытых портов:** Вся сеть работает только на исходящих соединениях (нет необходимости в Webhooks, белых IP или пробросе портов)
- **Мгновенная доставка:** Работа через сокет-события MTProto избавляет от необходимости писать парсеры и делать медленный http-поллинг
- **Готовность к Open Source:** Решение легко оформляется в публичный репозиторий на GitHub с инструкцией по генерации StringSession

## Решения по дизайну

| Решение          | Выбор                                                                     |
| ---------------- | ------------------------------------------------------------------------- |
| Метод публикации | `forwardMessage` — пересылка с сохранением атрибуции оригинального автора |
| Фильтрация       | Без фильтрации — все новые посты пересылаются без исключений              |
| Каналы-источники | Конфигурация через `.env` — список username/ID каналов                    |
| Маршрутизация    | Единый канал-агрегатор — все посты из всех источников идут в одно место   |

- [x] **Шаг 4: Reader / Producer — клиент (`src/producer/client.ts`)** — инициализация GramJS-клиента с `StringSession` из конфига, подключение к Telegram (`client.connect()`), обработка разрывов соединения (автопереподключение), логирование состояния подключения

- [ ] **Шаг 6: Writer / Consumer — бот (`src/consumer/bot.ts`)** — инициализация grammY-бота с токеном из конфига, функция `forwardMessage(chat_id, message_id, aggregator_channel)`, обработка ошибок (недоступность канала, удаление сообщения, права бота), повторные попытки при временных ошибках Telegram API (`FloodWait`), логирование успешных пересылок и ошибок
- [x] **Шаг 7: Точка входа (`src/index.ts`)** — инициализация логгера, загрузка конфигурации, запуск Producer (подключение клиента GramJS, запуск слушателя), запуск Consumer (инициализация бота grammY), связывание Producer и Consumer, graceful shutdown (обработка `SIGINT`/`SIGTERM`, корректное отключение клиента и бота)
- [ ] **Шаг 8: Lint и TypeCheck** — настроить ESLint + Prettier (или Biome) для TypeScript, убедиться что `npm run lint` и `npm run typecheck` проходят без ошибок
